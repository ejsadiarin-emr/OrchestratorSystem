    - _manual classification_ is needed for high-risk or non-idempotent packages
        - Package author/operator sets flags (idempotencyMode, riskLevel, retryabilityClass)

**Rollback & Retry Flow when a job failed mid-step**

1. Agent step fails -> sends `StepStatus(Failed, reasonCode)` to orchestrator.
2. Agent applies policy locally:
    - _transient_ failures (e.g. temporary failures, network drop, temp lock) + _retryable_ -> bounded retry (max 3 retries, 30/60/120 backoff).
    - non-retryable/high-risk -> fail step/job and stop.
3. If mutation already occurred and rollback contract exists:
    - agent performs rollback/restore locally (from snapshot) and reports rollback outcome.
4. Orchestrator records terminal state + audit trail and may trigger follow-up policy action (manual approval required, retry another node, etc.).
